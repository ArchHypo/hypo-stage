# Security policy

## Supported versions

We recommend running the **latest** `1.x` release from [npm](https://www.npmjs.com/org/archhypo) and applying Backstage security updates from the upstream project.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for undisclosed security problems.

- Use [GitHub Security advisories](https://github.com/ArchHypo/hypo-stage/security/advisories/new) for this repository (preferred), or  
- Contact the maintainers via a private channel if you cannot use GitHub.

Include steps to reproduce, affected versions, and impact if known. We aim to acknowledge reports within a few business days; timelines depend on severity and maintainer availability.

## Scope

This policy applies to the HypoStage monorepo (`@archhypo/plugin-hypo-stage`, `@archhypo/plugin-hypo-stage-backend`) and the documented deployment surface (REST API, database migrations, standalone demo configuration).
